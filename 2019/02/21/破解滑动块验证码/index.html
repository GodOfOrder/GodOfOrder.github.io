破解滑动块验证码

上篇文章有讲到使用selenium爬取Ajax数据, 其实selenium的功能不仅仅有这些, 注意到其API中有一个action_chains, 能够控制鼠标完成一系列操作. 于是想到了最近有遇到过的滑动模块的验证码, 如果能够操作鼠标移动滑块,是否可以软破解呢?

思路分析

需要实现的目标是将滑块拖动到图片缺口处,所以需要知道拖动的X轴方向距离,然后模拟人类的习惯设计拖动轨迹,故整体过程大致分为如下几步:

  • 从网页下载缺口图与拼图

  • 识别拼图缺口位置

    计算滑块拖动轨迹

  • 控制鼠标模拟滑块进行验证

页面分析

这里选取网易易盾为例, 图片的位置都很容易找到, 这里选取触发式的验证码类型,仔细观察一下滑块的生成:

发现滑块的初始位置距离图片边缘还是有一定距离的, 这里需要注意.

识别缺口位置

从图中可以看到, 滑块拼图与目标缺口只是饱和度不一样, 我们需要做的就是将整体图片进行遍历, 抓取与滑块大小相同的区域进行比较.

这里使用到opencv模块的 模板匹配 方法. 根据官方的描述, 该方法的原理就是在目标图像上滑动模板图像,并比较抓取的补丁区域与模板图像. 使用到cv2.matchTemplate()方法.

这里根据示例简单地进行比较:

    def match_Template(self,img,templ):
        image = cv2.imread(img, 0)    
        # imread(filename[,flags]) 读取一个图片文件

        img2 = image.copy()
        template = cv2.imread(templ, 0)
        w, h = template.shape[::-1]  # 宽,高
        img = img2.copy()
        method = eval('cv.TM_CCORR_NORMED')
        # eval:返回表达式的值,这里选取cv.TM_CCORR_NORMED: 归一化相关匹配法

        res = cv2.matchTemplate(image= img, templ=template, method=method)
        # templ: 搜索模板,必须小于img大小并具有相同数据类型

        min_val, max_val, min_loc, max_loc = cv2.minMaxLoc(res)
        # min_val,max_val: 返回指向最小(最大)值的指针
        # min_loc,max_loc: 返回指向最小(最大)位置的指针(2D下,故返回的是一个最小坐标)
        top_left = max_loc
        bottom_right = (top_left[0] + w, top_left[1] + h)

        cv.rectangle(img, top_left, bottom_right, 255, 2)  
        #第四个参数,255代表矩形颜色亮度
        #第五个参数,2代表构成矩形的线的厚度

        plt.subplot(121), plt.imshow(res, cmap='gray')
        plt.title('Matching Result'), plt.xticks([]), plt.yticks([])
        plt.subplot(122), plt.imshow(img, cmap='gray')
        plt.title('Detected Point'), plt.xticks([]), plt.yticks([])
        plt.suptitle('cv.TM_CCORR_NORMED')
        plt.show()

观察绘图的结果:

对matchTemplate()的6种匹配方式进行了尝试,发现”cv.TM_CCORR_NORMED”的匹配值更好,故这里选取该方法. 由上图可见, 匹配结果还是满足要求的, 那么需要做的就是获取X轴上偏移的距离.

调用cv2.minMaxLoc()方法, 将匹配的结果导入,返回值中包括上图所示的矩形边角的坐标值. 那么, 滑块需要偏移的X轴距离也清楚了.

滑块拖动轨迹

总的滑动距离已经获取, 那么现在需要模拟人的滑动轨迹, 通常的习惯一般是先加速滑动, 然后减速, 最后往回退一小段距离. 那么, 这里设定加速度与减速度为恒定值, 即先匀加速后匀减速.

位移公式为 s = V0 + (a * t^2)/2. 其中s为位移, V0为初始速度, a为加速度, t为经过这段位移的时间间隔. 代码示例如下:

    def modify_track(self,distance):
        distance = distance + 20
        vt = 0
        s = 0
        t= 0.2
        #取前4/5的距离为匀加速距离
        speed_dist = distance *0.8

        speed_track = []
        slow_track = []
        a = 1   #开始以加速度1加速
        while 1:
            offset = vt*t + 0.5 * a * (t ** 2)
            offset = round(offset)
            if s + offset > speed_dist:   #已经超过了加速的距离
                a = -2   #再以-2的加速度加速
            if s + offset > distance:
                speed_track.append(distance - s)
                break
            s = s + offset
            vt = vt + a
            speed_track.append(offset)

        slow_track = [-1,-1,-1,-2,-2,-2,-2,-2,-2,-2,-3]
        #list1记录需要前进的距离,list2记录需要倒退的距离
        return (speed_track,slow_track)

这里在原需要移动的距离基础上加上20, 是为了后续回退, 循环中计算每个t单位间隔内需要移动的距离.

这里注意要将每次的offset即位移设定为整数, 这样在后续控制鼠标完成行为链时就不会出问题.

函数返回值返回一个元组, 其中包含两个列表, 分别为匀加速过程的位移以及回退过程的位移.

控制鼠标模拟拖动滑块过程

这里使用到selenium API下的ActionChains, 这里注意, 之前分析图片的过程中是有发现滑块的初始位置距离图片边缘是有一段距离的, 经过测试, 将这个距离设定为8. 

#拖动滑块完成验证
def func1(self,track):
    btn = self.driver.find_element_by_xpath('//div[@class="yidun_slider"]')  #滑块位置
    ActionChains(self.driver).move_to_element(to_element=btn).perform()  
    ActionChains(self.driver).click_and_hold().perform()   #按住滑块

    for x in (x for x in track[0]):   #匀加速滑动
        ActionChains(self.driver).move_by_offset(xoffset=x,yoffset=0).perform()  
    for x in (x for x in track[1]):   #回退滑动
        ActionChains(self.driver).move_by_offset(xoffset=x,yoffset=0).perform()

    ActionChains(self.driver).release().perform()    #释放滑块

总结

滑动验证码的识别, 关键部分就是使用图像识别的知识(opencv的模板匹配)抓取匹配的部分, 然后计算出需要滑动的距离, 然后模拟人的拖动滑块习惯(先加速后减速再回退)进行轨迹的设计. 当然验证码不仅仅有这种方式, 后续也会尝试更多有趣的验证码.
